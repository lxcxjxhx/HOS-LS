# 常见绕过技术

## 概述

在渗透测试中，安全防御机制 (WAF、输入验证、认证等) 可能被绕过。了解这些技术有助于全面评估系统安全性。

## WAF 绕过

### SQL 注入绕过

1. **大小写混写**: `SeLeCt`, `UnIoN`
2. **内联注释**: `/*!SELECT*/`
3. **双重编码**: `%2527` (双 URL 编码)
4. **空白符替换**: `%09`, `%0A`, `%0D`, `%0C`
5. **特殊字符**: 反引号、方括号

### XSS 绕过

1. **事件处理器**: `onerror`, `onload`, `onmouseover`
2. **标签闭合**: 提前闭合 HTML 标签
3. **编码混合**: HTML 实体 + URL 编码
4. **数据 URI**: `data:text/html;base64,...`

## 输入验证绕过

### 文件上传绕过

1. **扩展名混淆**: `.php.jpg`, `.asp;.jpg`
2. **MIME 类型伪造**: 修改 Content-Type
3. **Magic Bytes**: 添加文件头签名
4. **路径遍历**: 在文件名中使用 `../`

### 内容长度绕过

1. **分块传输**: 使用 Transfer-Encoding: chunked
2. **多部分表单**: 分割大 payload

## 认证绕过

### JWT 绕过

1. **alg: none**: 使用无签名算法
2. **算法混淆**: RS256 -> HS256
3. **密钥爆破**: 常见弱密钥

### 会话管理绕过

1. **会话固定**: 固定会话 ID
2. **会话预测**: 可预测的会话令牌
3. **Cookie 篡改**: 修改 Cookie 值

## 防御建议

- **纵深防御**: 多层验证，不依赖单一防御
- **白名单优先**: 使用白名单而非黑名单
- **规范化输入**: 在处理前规范化所有输入
- **最小权限**: 服务使用最小必要权限
